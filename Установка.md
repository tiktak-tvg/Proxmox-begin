### Установка

#### Установка на Dibian
```bash
root@pve:~# hostnamectl
   Static hostname: pve
         Icon name: computer-vm
           Chassis: vm
        Machine ID: 334cd5e4243c312d8d0446ea0686f71243c304ad6
           Boot ID: bb6f6e243c304adf9df22419c912d8d0446ea0686
    Virtualization: vmware
  Operating System: Debian GNU/Linux 11 (bullseye)
            Kernel: Linux 5.15.74-1-pve
      Architecture: x86-64
root@pve:~#
```
Этот способ для опытных пользователей, которые хотят установить Proxmox на уже существующую систему Debian 12 (Bookworm)

> -Это не рекомендуется для новичков

#### Основные шаги:

Установите чистую Debian 12 (amd64) с минимальным набором пакетов (только "стандартные системные утилиты" и "SSH-сервер").

Настройте статический IP-адрес для вашего сервера и убедитесь, что имя хоста разрешается в IP-адрес (через файл /etc/hosts).

Добавьте репозиторий Proxmox VE в файл ``/etc/apt/sources.list.d/pve-enterprise.list``:
```bash
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" > /etc/apt/sources.list.d/pve-no-subscription.list
```
(Вы также можете добавить GPG-ключ для верификации пакетов).

Обновите список пакетов и установите Proxmox VE:
```bash
apt update && apt full-upgrade
apt install proxmox-ve postfix open-iscsi
```
После установки перезагрузите сервер. Если все прошло успешно, вы сможете загрузиться в режиме ядра Proxmox.

#### Настройка после установки
Сразу после установки рекомендуется выполнить несколько важных действий.

Первый вход в веб-интерфейс: Откройте браузер и перейдите по адресу ``https://<IP-адрес_вашего_сервера>:8006``. Войдите с логином root и паролем, заданным при установке.

Настройка репозиториев: Proxmox по умолчанию использует платный ``Enterprise`` репозиторий. Для тестовых сред лучше отключить его и включить бесплатный ``No-subscription`` репозиторий. 

Это можно сделать в веб-интерфейсе в разделе ``Updates → Repositories``.

Обновление системы: После настройки репозиториев выполните обновление системы через веб-интерфейс или консоль (``apt update && apt dist-upgrade``).

А лучше из консоли, отредактируем файлы репозиторий
```bash
root@pve:~# cat /etc/apt/sources.list.d/pve-enterprise.list
deb http://download.proxmox.com/debian/pve bullseye pve-no-subscription

root@pve:~# cat /etc/apt/sources.list
deb http://ftp.ru.debian.org/debian bullseye main contrib

deb http://ftp.ru.debian.org/debian bullseye-updates main contrib

# security updates
deb http://security.debian.org bullseye-security main contrib

Чтобы Docker работал корректно, вам нужно скачать официальный GPG-ключ и сохранить его по тому пути, который вы указали в файле репозитория.
mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg -o /etc/apt/keyrings/docker.asc
chmod a+r /etc/apt/keyrings/docker.asc

root@pve:~# cat /etc/apt/sources.list.d/docker.list
deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/debian bullseye stable
root@pve:~#
```

>[!Warning]
> - Если у вас возникла ошибка конфигурации пакетов MariaDB, которая блокирует любые дальнейшие действия apt (включая обновление и установку Docker).
> - Proxmox не использует MariaDB, поэтому эти пакеты можно смело удалить или принудительно исправить.
```bash
1. Создадим недостающий файл конфигурации
mkdir -p /etc/mysql
touch /etc/mysql/mariadb.cnf

2. Попробуем настроить пакет mariadb-common вручную
dpkg --configure mariadb-common

3. Если предыдущий шаг не помог, удалим проблемные пакеты принудительно
dpkg --remove --force-depends mariadb-common mariadb-client-core-10.5 libmariadb3 mariadb-client-10.5 mariadb-client

4. Очистим сломанные зависимости
apt --fix-broken install -y

5. Теперь обновление должно пройти без ошибок
apt update && apt upgrade -y
```
